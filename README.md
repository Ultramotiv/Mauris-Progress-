# Linux Folder Overview

This folder contains various subdirectories and files related to the Fairino Python SDK and its applications. Below is an overview of the contents:

## Subdirectories

### `ashutosh/`
Contains various Python scripts for robotic control and trajectory mapping. Key files include:
- `4.1.py`: Implements basic robotic movement.
- `4.2.py`: Enhances movement with additional parameters.
- `DMP_Traject.py`: Implements Dynamic Movement Primitives for trajectory generation.
- `drag_GCOM.py`: Handles drag mode with gravity compensation.
- `resistance_adaptive_newMap.py`: Adaptive resistance mapping for robotic control.

### `example/`
Contains example scripts demonstrating the use of the Fairino SDK. Key files include:
- `Test.py`: Basic test script for SDK functionality.
- `TestAngular.py`: Tests angular motion commands.
- `TestForceControlCommand.py`: Demonstrates force control commands.
- `TestMotionCommand.py`: Tests motion commands for the robot.

### `fairino/`
Contains the core Fairino SDK files. Key files include:
- `Robot.py`: Main Python interface for the Fairino robot.
- `setup.py`: Setup script for installing the SDK.

### `FT_Sensor/`
Contains scripts for force/torque sensor-based applications. Key files include:
- `a_active_SLS.py`: Active single-leg stance control.
- `a_guided_SLS.py`: Guided single-leg stance control.
- `adpt_speed_force.py`: Adaptive speed and force control.

### `Functional_exercise_Final/`
Contains finalized scripts for functional exercises.

### `Functional_exercises/`
Contains scripts for various functional exercises.

### `Game/`
Contains scripts for game-based rehabilitation applications.

### `RSA/`
Contains scripts related to robotic system analysis.

### `shreyash/`
Contains miscellaneous scripts by Shreyash.

### `Trash/`
Contains temporary or deprecated scripts.

### `UI/`
Contains scripts for the user interface of the robotic system.

## Files

### `trajectory_log.csv`
Logs of robotic trajectories.

### `.vscode/settings.json`
VS Code workspace settings.

For more details, refer to the individual README files in each subdirectory.
